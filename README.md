# my-first-app

ทดสอบหัดใช้ git clone url
gitignore ห้ามไฟล์ไหนขึ้น github
git pull ให้ใช้ก่อนเริ่มงานกับเพื่อน ดึงโค้ดล่าสุดจาก github
git log ถ้าอยากรู้ว่าที่ผ่านมาเราทำอะไรไปบ้าง ใครเป็นคนแก้ หรืออยากดูเลข ID ของการบันทึก (Commit Hash)
สร้าง Folder ให้ดูมืออาชีพ

การแยกกิ่งออกมาทำงาน
สร้างกิ่งใหม่: git branch feature/login
ย้ายไปทำงานที่กิ่งนั้น: git checkout feature/login
สร้างและย้ายทันที: git checkout -b feature/login

สรุป
บทที่ 1: รู้วิธีตั้งค่าเครื่องและกัน "ของต้องห้าม" ด้วย .gitignore
บทที่ 2: คล่องวงจร Add -> Commit -> Push ที่ต้องใช้ทุกวัน
บทที่ 3: รู้วิธีจัดบ้าน (Folder Structure) ให้เป็นระเบียบแบบ Backend Dev
บทที่ 4: รู้วิธี "แตกกิ่ง" (Branching) เพื่อทำงานร่วมกับทีมหรือลองของใหม่โดยไม่ทำโปรเจกต์พัง
🎓 สิ่งที่คุณทำได้ตอนนี้ (Skill Check):
✅ สร้าง Repository บน GitHub และ Clone ลงเครื่องเองได้
✅ จัดการไฟล์ความลับอย่าง .env ไม่ให้หลุดไปในที่สาธารณะ
✅ บันทึกประวัติการทำงานอย่างเป็นระบบ (Commit messages)
✅ แยกส่วนโค้ดเป็น Routes, Controllers, Models ตามมาตรฐาน
